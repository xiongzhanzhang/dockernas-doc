# 订阅应用模板

dockernas自带了一些常用的应用模板，但毕竟数量有限，用户可以在系统设置的订阅菜单下订阅第三方的应用模板，如下图所示：

# 模板格式介绍

第三方应用保存在nas存储目录的apps目录下，结构如下所示：

yi@DESKTOP-FQBM7BV:/mnt/g/nas/apps$ tree
.
└── demon
    ├── nginx
    │   ├── docker
    │   │   └── latest
    │   │       ├── nginx.conf
    │   │       └── template.json
    │   ├── icon.jpg
    │   └── introduction.json
    └── samba
        ├── docker
        │   └── latest
        │       └── template.json
        ├── icon.jpg
        └── introduction.json

可以看到demon仓库下有2个应用，分别为nginx和samba。每个应用底下的文件或目录作用如下：

  • icon.jpg为应用显示的图标
  • docker目录保存不同版本的模板，底下每个文件夹代表不同的版本
  • introduction.json为应用介绍信息，格式如下所示：
{
  "name": "nginx",
  "category": [
    "网络",
    "编程开发"
  ],
  "url":"https://www.nginx.com/",
  "summary": "Nginx是一个高性能的HTTP和反向代理web服务器"
}

应用模板最核心的文件在于template.json，这个文件告诉dockernas如何创建docker容器。nginx的模板文件如下所示：

{
    "imageUrl":"nginx:latest",
    "portParams":[
        {
            "prompt":"http访问端口",
            "key":"80",
            "value":"80",
            "protocol":"http"
        },
        {
            "prompt":"https访问端口",
            "key":"443",
            "value":"443",
            "protocol":"http"
        }
    ],
    "localVolume":[
        {
            "prompt":"配置文件",
            "key":"/etc/nginx/nginx.conf",
            "value":"",
            "name":"nginx.conf",
            "mountFile":true
        }
    ],
    "dfsVolume":[
        {
            "prompt":"证书文件目录",
            "key":"/ca",
            "value":"",
            "name":"ca"
        }
    ]
}

各配置项含义如下所示:

  • imageUrl：镜像地址
  • portParams：端口映射配置
  • localVolume：临时目录（即nas数据目录下的local目录）挂载配置
  • dfsVolume：共享目录（即nas数据目录下的dfs目录）挂载配置

更详细的配置参考源码的param.go文件。
