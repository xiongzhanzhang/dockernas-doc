# 安装docker

参考https://docs.docker.com/desktop/install/windows-install (opens new window)，使用WSL2后端，系统配置要求如下:

  • BIOS开启虚拟化
  • Windows10 64位(build 19043)或更高, 或者windows11 64位
  • 安装WSL2，在管理员模式下打开PowerShell，执行wsl --install，或者采用手动安装的方式，参考https://learn.microsoft.com/zh-cn/windows/wsl/install (opens new window)。

安装好后，系统托盘会出现Docker Desktop图标，可点开页面后查看docker服务状态是否正常。可进行如下配置：

  • 设置登录后自动启动docker
  • 配置镜像加速，完整配置如下所示：
{
  "builder": {
    "gc": {
      "defaultKeepStorage": "20GB",
      "enabled": true
    }
  },
  "experimental": false,
  "features": {
    "buildkit": true
  },
  "registry-mirrors": [
    "https://registry.docker-cn.com",
    "http://hub-mirror.c.163.com",
    "https://docker.mirrors.ustc.edu.cn"
  ]
}

# 安装dockernas

控制台执行如下命令：

docker run -d --name dockernas --restart always -p 8080:8080 -v /var/run/docker.sock:/var/run/docker.sock -v G:\nas:/home/dockernas/data xiongzhanzhang/dockernas

注意8080端口不能被占用，G:\nas为nas系统存储数据的目录，需替换为自己机器实际存储数据的目录。可在Docker Desktop查看容器是否顺利启动，浏览器访问http://localhost:8080 (opens new window)，账号密码保存在nas数据目录下的config.json文件中。

nas数据目录结构如下所示：

yi@DESKTOP-FQBM7BV:/mnt/g/nas$ tree -L 1
.
├── apps
├── config.json
├── dfs
├── local
├── logs
└── meta

各目录作用如下：

  • apps目录保存第三方应用模板，注意不要放置其它文件到该目录下。
  • dfs保存的是应用数据，部署应用的时候，可能会需要选择这个目录底下的某个文件夹作为数据保存目录。
  • local目录保存临时的应用数据，应用删除这里对应的应用数据也会删除掉。注意，部署应用的时候如果数据目录选项留空，会将数据保存到local目录。
  • logs为系统日志，报错时可在这里查看原因。
  • meta为元数据目录。

# 安装filebrowser

找到filebrowser应用，点击进行配置，如下所示：

基本配置解释如下所示：

  • 实例名为实例名字，不能重复、不能包含中文
  • 注备可不填。一个应用可以部署多个实例，可以使用注备标准不同应用的用途
  • 仅本地访问开启后，应用的网络服务只能在本机访问，这样可以增加网络安全性
  • 版本为应用的版本，一般选择latest

上述配置为部署所有应用通用的配置，部分应用还有自定义的配置，filebrowser的自定义配置解释如下所示：

  • web访问端口：指filebrowser在本机的访问端口，主要访问端口不能和其他实例冲突，否则会无法部署
  • 管理的目录：对应nas数据目录下，dfs文件夹的子目录，图中选择了根目录，表示管理dfs整个目录。为了安全性，尽量最小化每个应用实例使用的目录，避免不同实例之间冲突，也能提升数据安全性
